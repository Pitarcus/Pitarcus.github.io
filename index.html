Everything, Everywhere, On a Roll! - UJI Game Jam 2024

A 48 hour jam game where you roll a ball and explore a cute garden. I developed the textures and models with Blender, TreeIt and Subtance Painter. Created sounds using Ableton Live and implemented them with FMOD. I used systems I had created previously for the control of UI animations, FMOD control and day/night cycle. Modified a free toon shader asset to fit my needs and used a Subsurface Scattering subgraph I have created previously for the tree leaves.

Read More Itch.io

The movement was created using an scalable 3rd person physics controller I had developed previously. The camera controls were also created previously and use Cinemachine's FreeLook Camera.

The main mechanic of sticking objects to the ball is pretty simple and uses a leveling system to allow or deny the player sticking certain objects.

You can check some of the most notable scripts here:

PlayerMovement Day/Night Cycle UIAnimator PostprocessTweener

All of the assets where modeled or modified in Blender, taking into account the scale between them and the player, making it a 10:1 scale for a character that is 2cm tall.

I made sure the downloaded models (pine cone, phone, and key) have a free personal and commercial license. Also, reduced tri-count and left the UVs to Substance Painter.

They were then textured inside Substance Painter for the swiftest pipeline effort. The UVs are handled by Substance which is not ideal, but it cut a lot of time.

Finally, they are assembled into different prefabs so instancing them in the world is optimized and easy using Polybrush. They use a free Toon Shader and also a custom stylized plastic shader that I created for the bottle and pen materials.

The environment has been modelled and UV unwrapped in Blender and painted in Substance Painter, using default and customs materials that I have created in Designer. The trees have been generated using TreeIt, the free vegetation generation tool.

The main lit shader for URP is used for all of the objects, with the Unity URP preset for exporting textures in Painter. The water shader was created for 'Blue Blood' and has been reused here. A custom subsurface scattering subgraph developed for Wild Foliage (my grass and vegetation system) and a wind subgraph (global wind) from the same project is used on the tree leaves.

For the tree leaves, now that the Jam has finished, I decided to make them more honest to real life and get actual leaves from the orange trees, as they are very common in my city. I scanned them, used Photoshop to cut and process some leaves and used Materialize to generate all the necessary PBR textures.

There are two types of leaf cards, one is a branch, also generated in Tree it, in order to increase density and the other one is a simple leaf as this allows for more detailed animation.

Grass and dynamic wind - Final Degree Project 2023 / Ongoing

The final project for my Videogame Design and Development bachelor's. It features tools for customized rendering of grass with the help of compute shaders, which I used to make similar grass to Breath of the Wild. A fluid simulation based wind system creates dynamic interaction with motors, inspired by God of War. You can check out some of the development process on my Twitter account!

Read More Video Trailer

My Bachelor's Thesis was centered around making an optimized and useful grass tool. Taking into consideration the best looking grass in videogames (Breath of the Wild and Ghost of Tsushima come to mind), they use model based blades, so that is what I wanted to achieve.

In these slides we will take an overview of the system, but a thorough explanation can be found in my thesis' report.

Bachelor Thesis

Also, some of the work and some short videos of the progress can be found in my twitter account.

Twitter Account

The bike featured in the image was also created by me using Blender.

Learning GPU Instancing in Unity and Compute Shaders, I was able to achieve a flexible grass rendering system that scales well thanks to optimizations (world partitioning using a quadtree and frustum culling) and is highly customizable. The system is used in my Jam game 'Everything, Everywhere, On a Roll!' and people have not reported problems with the grass yet!

As one of the current focus of the project is making it as user friendly as possible, I am working on streamlining the use of the plugin: creating prefabs, fixing editor bugs and organizing the project's folders. Here is a link to the repository of the plugin:

Repository

Another focus of this project was making the grass look appealing, so a complex vertex and fragment shader had to be developed. This was achieved using Shader Graph, which offers the opportunity to create modular pieces that can be linked together, reused and improved separately, which is amazing!

The vertex shader features static wind (general wind), dynamic wind; rotation, scale, bend and their randomizations; object displacement and camera awareness so the view is not obstructed.

The fragment shader uses parameterized colors for the top and the bottom of the blade, a texture for color variation of the tip and fake subsurface scattering to make it look more alive.

The 'static' wind (general wind) works in a global scope and affects all shaders that use the static wind module (or the global shader variable). This way we can have a coherent wind that can be sampled by any object in the scene! It is also modifiable and customizable.

The dynamic wind uses a fluid simulation that is computed in an enclosed space around the player. The current implementation is not 100% accurate nor fully optimized, but it works in a good enough way on high end devices. It is also modular and can be added to any other shader in shader graph thanks to a dynamic wind master.
